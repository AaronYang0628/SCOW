# 编写集群信息配置文件

通过`clusters.{json|yml|yaml}`文件定义集群的基本信息，如集群的节点数、内存、核心、GPU数等信息，将其放在`config`目录下，并将其映射各个容器的`/etc/scow`目录中。

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="config/clusters.yml" default>

```yaml
hpc01: # 集群ID
  displayName: hpc01Name # 集群的名称
  partitions: # 集群的各个分区
    compute: # 分区名
      nodes: 28 # 分区内节点数
      mem: 7500 # 单节点内存数量，单位M
      cores: 2 # 核心数
      gpus: 0 # GPU数
      qos: # QOS的数量
      - low
      - normal
      - high
      comment: '' # 这个分区的备注信息
    GPU:
      nodes: 1
      mem: 262144
      cores: 48
      gpus: 8
      qos:
      - low
      - normal
      - high
      comment: ''
```

</TabItem>
